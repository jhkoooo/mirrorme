`C:\GitHub\mirrorme\` 프로젝트(MirrorMe)에서 작업할 때, **사용자 프롬프트로 인해 실제 파일이 수정·생성된 경우** 반드시 `C:\GitHub\mirrorme\CHANGES.md`에 항목을 추가한다. (옛 위치 `C:\Users\koo\Desktop\MirrorMe\`는 백업/참조용)

**Why:** 사용자가 어떤 지시로 어떤 변경이 일어났는지 시간순으로 추적하고 싶다고 명시적으로 요청함 (2026-04-09). PROJECT.md는 "현재 상태"를 보여주는 문서이고, CHANGES.md는 "왜 이렇게 됐는지(프롬프트→변경)"의 이력 문서로 분리.

**How to apply:**
- **기록 대상**: 코드 파일(.html/.js/.swift/.css 등) 또는 PROJECT.md/메모리 등 프로젝트 문서가 실제로 수정·생성된 경우만.
- **기록 제외**: 단순 질문(예: "Mac 없이 앱 만들 수 있어?", "줌 몇 배까지 돼?", 호스팅 방법 안내 등) 처럼 파일 수정이 없는 응답은 기록하지 않음.
- **포맷**: 날짜 섹션 아래 번호 매기고, 다음 4개 필드 — 프롬프트(가능하면 verbatim, 길면 앞부분 + ...), 수정/생성 파일 목록, 변경 내용 요약.
- **타이밍**: 코드 수정 작업의 마지막 단계로 함께 처리. 별도 사용자 요청이 없어도 자동으로.
- 단, "지금 작업한 거 CHANGES에 기록할까?"같은 메타 확인은 안 해도 됨 — 그냥 조용히 추가.
