# MyStyle (구 MirrorMe)

거울/스타일 관리 앱. "상대방이 보는 내 모습"으로 시작해, v3.3에서 앱 이름을 **MyStyle**로 공식 리브랜딩. 거울 모드는 메인 기능 중 하나로 남고, OOTD 기록이 중심 축.

> **이름 이력**: v1~v3.2까지 `MirrorMe`. v3.3부터 `MyStyle`. GitHub 저장소·URL(`jhkoooo.github.io/mirrorme/`)·IndexedDB(`MirrorMePhotos`)·GoatCounter 대시보드는 연속성을 위해 MirrorMe 이름 유지.

**사용자 관점**: 본인의 스타일을 기록하고 AI가 밸런스를 판단해준다. 다른 사용자의 데이터는 일절 노출되지 않는다 (사용자끼리 공유 X).

**개발자(관리자) 관점**: 동의받은 사용자 데이터를 수집·열람해 패션 트렌드 분석·서비스 개선에 활용한다. 관리자 대시보드는 본인만 접근 가능.

## 컨셉

일반 거울이나 iOS 기본 셀카 프리뷰는 **좌우가 반전**되어 보인다 (내가 오른손을 들면 화면 속에서도 오른쪽에 손이 올라옴 = 실제로는 왼쪽). 이 앱은 그 반전을 해제해서 **다른 사람 눈에 비치는 내 모습**과 동일하게 보여준다.

## 빌드 전략 — PWA로 결정 (2026-04-08)

원래 SwiftUI 네이티브 앱으로 시작했으나, 빌드에 Mac/Xcode가 필요해 **PWA(Progressive Web App)** 로 전환했다.

- 장점: Mac 0원, Apple Developer 계정 0원, Windows에서 작성·배포 가능, 즉시 아이폰에 설치
- 단점: 잠금화면 위젯·푸시 같은 고급 기능 약함 → 거울/패션 기록 용도엔 영향 없음
- 기존 SwiftUI 코드는 [`MirrorMe/`](MirrorMe/) 폴더에 보관 (나중에 Mac 생기면 v∞에서 네이티브 전환 옵션)

## 장기 비전 — MyStyle (2026-04-09 기록)

거울 앱에서 출발했지만, 장기적으로는 단순 거울이 아니라 **본인 스타일 매니저**가 목표다. 이름도 결국 `MirrorMe`보다 `MyStyle`이 더 어울린다.

### 탭 구조 (구상)
- **거울 탭** — 현재 v1.5의 "상대방이 보는 나" 기능 (지금까지 만든 것)
- **OOTD 탭** — 날짜별로 오늘의 옷 사진/메모 기록
- **앨범 탭** — 모든 촬영 사진을 한 번에 보기 (현재 v2의 갤러리)
- **분석/통계 탭** — 자주 입은 옷, 색상 분포, 스타일 추이 등
- 그 외 분야별 분류 탭들

### 핵심 기능 비전
- **날짜별 OOTD 기록** — 사진 + 날짜 + 태그(상의/하의/신발/액세서리) + 메모
- **AI 스타일 밸런스 판단** — 사진을 분석해 색상·실루엣·아이템 조합이 균형 잡혔는지 판정. (예: Claude Vision API, Google Vision 등)
- **회고/추이** — "이 옷 마지막에 언제 입었지?", 월간 캘린더 뷰, 자주 입은 조합

### 배포 정책

- **공개 배포 O** — App Store 또는 PWA URL로 일반 사용자에게 배포
- 사용자끼리는 데이터 공유 안 됨 — 각자의 사진/기록은 본인만 봄
- **관리자(개발자)** 만 별도 admin 화면에서 전체 데이터 열람 가능
- 데이터 수집은 **반드시 가입 시 동의 화면**으로 합법화 (개인정보처리방침 + 명시적 동의 체크박스)

### 백엔드 구조 (Mac/배포 시점에 구축)

```
사용자 앱 ──업로드──> Firebase
                      ├─ Auth          (사용자 로그인)
                      ├─ Storage       (사진 파일)
                      └─ Firestore     (날짜·태그·메모 메타데이터)
                              │
                              │ 관리자 권한만 read 허용
                              ↓
                      Admin 대시보드 (별도 /admin 페이지)
                      - 본인만 로그인
                      - 모든 사진 열람·필터·분류
                      - AI 자동 태깅 (Claude Vision API)
                      - 통계·트렌드 집계
```

- **추천 스택**: Firebase Auth + Storage + Firestore (무료 티어로 ~100명 ×월 10장 정도 충분)
- **Admin 권한 분리**: Firebase Auth custom claims로 admin 역할 부여, Firestore Security Rules로 일반 사용자는 본인 UID 데이터만 read/write 가능
- **AI 분석**: Claude Vision API에 사진 + "스타일 밸런스 평가" 프롬프트 전송 → 색상·실루엣·아이템 조합 점수 회신

## 아이폰에 설치하는 방법

> ⚠️ **HTTPS 필수.** iOS Safari는 `localhost`가 아닌 한 HTTP에서 카메라(`getUserMedia`)를 허용하지 않는다. 반드시 아래 호스팅 방법 중 하나로 HTTPS URL을 만들어야 한다.

### 방법 A — GitHub Pages (가장 추천, 무료, 영구) ⭐

1. [github.com](https://github.com)에서 새 저장소 생성 (예: `mirrorme`). public으로.
2. `docs/` 폴더 안의 파일들을 저장소 루트에 업로드 (드래그앤드롭 가능).
3. 저장소 → **Settings → Pages** → Source: `Deploy from a branch` → Branch: `main` / `/ (root)` → Save.
4. 1~2분 후 `https://<본인아이디>.github.io/mirrorme/` 주소가 활성화됨.
5. 아이폰 **Safari**로 그 주소 접속 (Chrome 말고 꼭 Safari).
6. 하단 공유 버튼 → **"홈 화면에 추가"** → 추가.
7. 홈 화면의 MirrorMe 아이콘 탭 → "시작하기" → 카메라 권한 허용 → 끝.

### 방법 B — 같은 Wi-Fi에서 임시 테스트

GitHub 올리기 전에 빠르게 확인만 하고 싶다면:

1. PC에서 `docs/` 폴더로 이동 후 간이 서버 실행:
   - Python: `python -m http.server 8000`
2. 같은 Wi-Fi의 아이폰 Safari에서 `http://<PC의 IP>:8000` 접속
3. **단, 이 방법은 카메라가 동작하지 않는다** (HTTPS 아님). UI 미리보기 용도로만.
4. 카메라까지 테스트하려면 [`ngrok`](https://ngrok.com) 같은 터널링으로 HTTPS URL을 받아야 함:
   - `ngrok http 8000` → 발급된 `https://xxxx.ngrok-free.app` 주소를 아이폰 Safari로 접속

→ **장기적으론 방법 A 권장.** 한 번 올려두면 코드 수정할 때마다 push만 하면 된다.

### 방법 C — Netlify Drop (가장 빠름, 가입 없이 즉시 HTTPS)

1. [app.netlify.com/drop](https://app.netlify.com/drop) 접속
2. `docs/` 폴더를 브라우저로 드래그앤드롭
3. 즉시 `https://<랜덤이름>.netlify.app` URL 생성됨
4. 아이폰 Safari로 접속 → 홈 화면에 추가

가입 없이도 임시 사이트가 만들어진다 (영구 보관하려면 무료 가입 권장).

## 사용 주의사항

- 반드시 **Safari**로 열어야 "홈 화면에 추가"가 보인다. (Chrome iOS는 안 됨)
- 첫 진입 시 카메라 권한 팝업이 뜬다. 거부했다면 설정 → Safari → 카메라 → 허용으로 변경.
- 홈 화면 아이콘으로 들어가면 주소창 없이 풀스크린으로 뜬다 (앱처럼).

## 기술 메모

- **왜 CSS `transform: scaleX(-1)`이 없는가?** HTML `<video>` 요소는 카메라 스트림을 **있는 그대로** 표시한다. 즉, 기본 상태가 이미 "상대방이 보는 나"다. iOS 네이티브 앱이라면 거꾸로 미러링이 기본이라 꺼야 했지만, 웹에서는 아무것도 안 하면 된다.
- **iOS Safari는 사용자 제스처 후에만** `getUserMedia`가 안정적으로 동작 → 그래서 "시작하기" 버튼을 둠.
- **`playsinline` 필수** — 안 붙이면 iOS가 비디오를 자동으로 풀스크린 플레이어로 띄워버림.
- **HTTPS 필수** — `localhost` 외의 HTTP에서는 카메라 API 자체가 차단됨.
- **접속 통계** — GoatCounter (쿠키 없음, 개인 식별 안 함). 대시보드: `https://mirrorme.goatcounter.com`. `index.html` 하단의 `<script>` 한 줄로 동작.
